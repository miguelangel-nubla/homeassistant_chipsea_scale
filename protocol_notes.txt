Chipsea Scale Protocol Documentation
====================================

Packet Structure
----------------
19 bytes total, using first 8 bytes:

Byte 0: 0xCA (header)
Byte 1: 0x10 (command/status)  
Byte 2: Unknown/unused
Byte 3: Status byte (unit, decimals, sign, stability)
Byte 4: Unknown/unused
Bytes 5-6: Weight data (16-bit big-endian)
Byte 7: Unknown/unused

Status Byte (byte 3) - Bit Layout
----------------------------------
Bit 7: Sign (1 = negative, 0 = positive)
Bits 6-3: Unit type (4 bits)
Bits 2-1: Decimal places (2 bits)  
Bit 0: Stability (1 = stable, 0 = changing)

Unit Type Encoding (bits 6-3):
- 0000: Grams
- 0011: Ounces
- 0110: Pounds  
- 1000: Kilograms

Weight Processing
-----------------
1. Extract raw weight from bytes 5-6 (big-endian)
2. Scale by decimals: weight = raw ÷ (10^decimal_bits)
3. Apply sign from bit 7 of status byte
4. Result is weight in native unit

Examples:
- Status 0x46 (01000110): kg, 3 decimals, positive, unstable (bit 0 = 0)
- Status 0x47 (01000111): kg, 3 decimals, positive, stable (bit 0 = 1)  
- Status 0x1C (00011100): oz, 2 decimals, positive, unstable (bit 0 = 0)
- Status 0xC7 (11000111): kg, 3 decimals, negative, stable (bit 0 = 1)

Sample Calculation:
Raw 0x0808 (2056), status 0x46 → 2056÷1000 = +2.056 kg
